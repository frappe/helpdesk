Fixes #2629

## Description
Adds the ability to react to comments with multiple emojis. This allows users to acknowledge comments without creating additional replies, reducing ticket clutter.

## Screenshot
![Comment Reactions Feature](https://github.com/user-attachments/assets/cf5eb279-e407-4a21-88d7-3c5ace60cb47)

## Problem Solved
When someone is tagged in a comment for informational purposes, they previously had two options:
1. Add a new comment to acknowledge (clutters the ticket)
2. Read without responding (no acknowledgement)

Now users can simply react with emojis to acknowledge they've seen the comment.

## Changes

### Backend
- New child table DocType `HD Comment Reaction` to store reactions inside comments
- Added `reactions` Table field to `HD Ticket Comment`
- API: `toggle_reaction(comment, emoji)` - add/remove reaction
- API: `get_reactions(comment)` - get reactions grouped by emoji with user info
- Notification sent to comment author when someone reacts
